# 预算管理小程序

这是一个基于微信小程序云开发 quickstart 演进中的预算管理项目，当前以原生微信小程序为主，先完成个人预算 MVP 的页面、金额工具和本地服务边界。云函数目录仍保留 quickstart 结构，后续再按计划接入云数据库、云函数或迁移到目标技术栈。

## 项目定位

项目目标是构建一个从个人预算逐步扩展到组织预算的预算管理系统，核心流程是：

```text
创建预算 -> 记录流水 -> 扣减预算 -> 分析优化
```

当前仓库已经具备个人预算的本地可用闭环：

- 在“我的”页面设置和切换月度预算。
- 在“记账”页面新增或编辑消费记录。
- 在首页查看总预算、已使用、剩余额度、预算状态和近期消费。
- 在“明细”页面按消费类型筛选、编辑或移除订单。
- 在“分析”页面查看分类排行、近 7 日趋势和预算建议。
- 本地保存昵称、头像、预算、消费流水和预算变更记录。

## 当前技术栈

- 小程序：原生微信小程序，WXML + WXSS + JavaScript。
- 云能力：微信云开发 quickstart，当前仅初始化云环境，业务数据暂未接入云数据库。
- 本地状态：`wx.getStorageSync` / `wx.setStorageSync`，脱离微信运行环境时回退到内存状态。
- 金额计算：使用整数“分”作为内部计算表示，对外统一保存和展示为两位小数的“元”金额字符串。

目标技术栈（UniApp、Vue 3、TypeScript、Cloudflare Worker、Hono、D1 等）目前属于后续迁移方向，不是当前运行基线。

## 运行方式

1. 使用微信开发者工具导入本仓库根目录。
2. 确认 `project.config.json` 中的目录配置：
   - `miniprogramRoot`: `miniprogram/`
   - `cloudfunctionRoot`: `cloudfunctions/`
3. 如果需要使用真实云环境，在 `miniprogram/app.js` 或环境配置中填入对应云环境 ID。
4. 在微信开发者工具中编译运行小程序。

当前预算业务数据存储在本地缓存中，主要键名如下：

- `budget_flow_state_v1`：预算、消费流水、预算变更记录和当前查看月份。
- `budget_flow_user_profile_v1`：昵称、头像等用户资料。

## 关键业务规则

- 金额单位统一为“元”，字段优先使用 `_yuan` 后缀，例如 `amount_yuan`、`total_amount_yuan`、`used_amount_yuan`。
- 金额进入业务逻辑前必须校验并格式化为两位小数字符串，例如 `100.00`。
- 不直接使用 JavaScript 浮点数做金额加减，当前通过整数“分”进行精确计算。
- 预算状态阈值：
  - 正常：使用率 `< 80%`
  - 预警：使用率 `>= 80%` 且 `<= 100%`
  - 超预算：使用率 `> 100%`
- 新增或编辑消费时，如果会导致超出预算，会阻断提交并提示用户先调整预算。
- 每次新增、编辑或移除消费都会写入本地 `mutations` 记录，便于后续扩展为可审计预算变更。
- 仅当前月份可编辑；每月前两天允许编辑上月记录；未来月份和更早月份默认只读。

## 开发约定

- 页面文件只做生命周期、事件处理和 UI 状态协调。
- 金额校验、预算计算、持久化编排放在 `miniprogram/utils/` 或 `miniprogram/services/`。
- 新增页面、组件或服务前先搜索现有实现，优先复用当前边界。
- 预算数据属于私密财务数据，后续接入云端时必须按用户或组织作用域隔离。
- 不要在没有明确迁移需求时引入 UniApp、Vue、Cloudflare Worker 或 D1 相关模块。
- 不要修改 `project.private.config.json`，除非任务明确涉及本地微信开发者工具配置。

## 手动验证

当前仓库没有可直接运行的自动化测试。修改预算相关功能后，建议在微信开发者工具中按以下路径验证：

1. 进入“我的”，设置本月预算，例如 `1000.00`。
2. 进入“记账”，选择消费类型并保存一笔消费。
3. 回到首页，确认已使用金额、剩余额度、预算状态和近期消费更新。
4. 进入“明细”，按类型筛选，长按记录验证编辑和移除。
5. 进入“分析”，确认分类排行、近 7 日趋势和建议文案正常。
6. 尝试录入超过剩余额度的消费，确认会提示超预算并阻断保存。
